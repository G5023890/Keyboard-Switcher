# Third-Party Notices

This file documents third-party data and externally supplied assets included in Keyboard Switcher `v0.75`.

## Russian Frequency Data

Bundled file:

```text
KeyboardSwitcher/Resources/russian-frequency-50000.txt
```

Source data:

- University of Leeds Russian Internet Corpus frequency list: http://corpus.leeds.ac.uk/frqc/internet-ru.num
- Cleaned repository by William Hingston: https://github.com/hingston/russian

License:

- Creative Commons Attribution 2.5: https://creativecommons.org/licenses/by/2.5/

Source notice from `hingston/russian`:

```text
Data files are derived from the University of Leeds Corpus, found at
http://corpus.leeds.ac.uk/frqc/internet-ru.num, and distributed under
the Creative Commons Attribution 2.5 license. Corpus editing and cleanup
by William Hingston.
```

Modification note:

Keyboard Switcher includes a compact subset/reformatted derived word-frequency list for local scoring.

## English Common-Word Data

Bundled file:

```text
KeyboardSwitcher/Resources/english-common-5000.txt
```

Source:

- Michael Wehar Public-Domain-Word-Lists: https://github.com/MichaelWehar/Public-Domain-Word-Lists
- Source file: `5000-more-common.txt`

License/status:

The source repository README describes `5000-more-common.txt` as generated by Michael Wehar and released into the public domain.

Source notice:

```text
5000-more-common.txt is a list containing 5000 words that have been marked
as common. This list was generated by Michael Wehar and is released into
the public domain. The source README also notes that the list was generated
while utilizing OPTED.
```

## Evaluated But Not Bundled

The following resources were considered during development but are not bundled in `v0.75`:

- `first20hours/google-10000-english`: not included because its license notice cautions against commercial use without licensing from the Linguistic Data Consortium.
- `dwyl/english-words`: not included because the app does not currently need a very large English word list.
- `rspeer/wordfreq`: not included because the project currently avoids heavy package data/runtime dependencies.
